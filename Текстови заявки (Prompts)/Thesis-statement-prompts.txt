Въведение - Към всеки от разглежданите големи езикови модели за свободна употреба е изпратена заявка (т.н. “prompt”) който има за цел да тества генеративните способности на моделите за разбиране и изпълняване на текстова заявка. Задачата на всеки модел е да генерира базов клас използвайки програмният език Java и да предостави реализация на линейна структура от данни - единично свързан списък с операция за добавяне на нов елемент, 
премахване на елемент и принтиране на резултата в конзолата. Веднага след това всеки модел е използван за да се генерира документация по генерираният от него код. 

Стъпка (1) - Текстова заявка (“prompt”) за генериране на Java код за едносвързан списък:
“Write a Java class for a singly linked list with methods to insert an element at the end, delete an element by key, and print the list.” 

Стъпка (2) - Текстова заявка (“prompt”) за генериране на Java документация по примерен код:
“Generate a comprehensive confluence-format documentation for the following Java code: <Code Generated By Each Model>” 
Където <Code Generated By Each Model> се заменя с генерираният от стъпка 1 резултат от всеки модел и след това се изпраща
